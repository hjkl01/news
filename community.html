  • [编程] 邀请各路编程语言做 benchmark，进行性能比较
    社区   V2EX   2024-10-05 09:52:49

    介绍

    这个库使用 nix 包管理系统管理各种语言的开发环境，它同时集成了一些基础的性能测试代码。这样做的一个优点是，你可以把这个库 clone 下来在自己的机器上做测试，而不必担心扰乱原有的环境。

    目前库里面有两种语言四种环境：

    1. javascript/bun
    2. javascript/nodejs
    3. javascript/quickjs
    4. scheme/chezscheme

    目前的 benchmark 主要集中于基础的向量操作、字符串操作、递归操作，考察的主要是各种语言环境的优化能力和优化结果。

    在 v 站发帖，主要是希望有朋友能来帮忙做如下工作：

    1. 贡献 Go 语言的相关 benchmark ，而且一定要和已有的代码对标，就是原来的代码是什么风格，那么你 go 应当用同样的风格去写代码。原来是用递归实现的，你不能强行拆成用循环实现；
    2. 贡献各种国产编程语言的 benchmark ，要求同 go 。不过很多国产编程语言在 nix 包管理器里面没有打包，大概需要先通过 flake.nix 实现一个自己的打包。

    补充

    这个项目的不足在于 3 点：

    1. 其实 benchmark 主要用递归实现，可能对于一些语言的优化不友好。但是这里主要考虑的是 benchmark 的可比性，因为递归是标配。你用这个语言的高级特性，我用那个语言的高级特性，那就没法好好对比了。
    2. 目前测试使用的是 linux 一般标配的 time 命令（注意，不是 bash 的内置关键字）。这实际上测试的时候，就不可避免受到语言环境的启动时间影响——我们测的其实是各种环境跑脚本的性能，只不过有些脚本跑的时间长，启动时间可以忽略。这对于 Java 之类的语言可能不友好，而对于 C 之类的语言似乎又有优势。
    3. 目前测试没有加入 C 和 rust 之类的语言，因为它们一般性的需要编译一个可执行文件，这里面的时间消耗的组配和已有编程语言不可比。

    支持的 OCR 库包括：Tesseract （ Apache 2.0 许可证）、EasyOCR （ Apache 2.0 许可证）、WeChat OCR （未知许可证，仅支持 Windows ）、Surya （ GPL 3.0 许可证）以及 RapidOCR （ Apache 2.0 许可证）。

    使用示例

    from my_little_ocr import get_engine_instance

engine = get_engine_instance("rapidocr")
result = engine.ocr("img1.png")

print(result.to_list())

    切换引擎只需更改 get_engine_instance() 中的引擎名称。欢迎大家来贡献——快来看看吧！

    GitHub 项目地址： https://github.com/X-T-E-R/my-little-ocr

    安装命令：pip install my_little_ocr

    一些心得：

    1. 需要建立起对经济周期的认知，不同的经济周期有不同的投资选择。理清美元、美债/债券、黄金、股市他们之间的关系。有很多的宏观经济指标可以辅助判断经济周期。富途的 [一学就懂的宏观经济课] 挺不错，还有曼昆的 [经济学原理] —— 宏观经济学分册。
    2. 美股可以整价值投资，A 股只适合投机，港股/中概股是比较优质的投机股。国内的股市是政(诈)策(骗)市，政府主导，在政策市下讨论哪个股票有价值是没有意义的，有意义的指标是共识——大家认为好才是真的好，自己认为好并不一定好。
    3. 能看到的数据是意味着过去的情况，并不代表未来；自己需要需要结合当下的信息来给出未来一个预期；如果未来与预期不符需要及时修正。需要分清当前的市场在由基本面(政策、宏观数据、信息等)中的哪个指标来主导：在主导指标不变的情况下，前一段时间的股票在猛涨，当前在狂跌，则意味着买入机会。
    4. 自己的预期需要结合别人的预期一起看。这个别人指的是：1. 经过历史验证了正确率的。2. 使用了真金白银去市场投票了的。3. 远离一分钱不出的口嗨党。
    5. 大行情下不存在什么信息差，缺失的是自身的判断与下决定的勇气。学会反人性——勿追涨杀跌，面对猛跌的时候要么反思，要么有加仓的觉悟。
    6. 要相信 gov 的底线是保持经济可以(而非正常)运转——这也不相信的话就没必要在这片土地上待了吧。穷人没饭吃要掀桌子，富人怎么都是赢，全球的 gov 都在押中产不敢掀桌子。
    7. 根据风险偏好的不同使用适当的措施对冲风险。

    提供几个工具/方法辅助判断：

    • 美联储降息预测: https://www.cmegroup.com/markets/interest-rates/cme-fedwatch-tool.html
    • 长期期权/期货的价格还是比较稳定的，意味着大家未来对这个东西价值的预期。比如美元人民币汇率期货： https://sc.hkex.com.hk/TuniS/www.hkex.com.hk/Products/Listed-Derivatives/Foreign-Exchange/USD-CNH-Futures
    • 关于近期 A 股的走势观察：USD/CNH 实时汇率、北向资金流入、交易量、几个 ETF——FXI,KWEB,MCHI,ASHR

    股市里没有 100%的赢家，是根据已知信息去押大小。有的人喜欢高收益，看不上必赢的丁点收益，就去押小概率事件；有的人愿意牺牲流动性去搏低风险高收益…… 不同的人有不同的选择。找到合适的才是最好的。

    很多东西，不去尝试永远不知道行不行；经过事实检验的观点才是好的观点. 主观的事情，一千个人有一千个哈姆雷特，但是客观的事情只有一个哈姆雷特。如果你的观点经常与未来事实不符，要么通过反思自己来修正观点，要么放弃传播给他人，除非你本就怀有恶意. 万事无绝对，风险与收益并存，从来不存在低风险、高收益的东西。

    关于这几天的股市，我挺佩服上周一与周二加仓的人，上周一之前买入的人只是存在一个会涨起来的信念。 让我更印象深刻的是中国人还是钱多、喜欢赚钱，哪里有钱赚就一窝蜂拥上去，完美符合普通人、有钱赚、不排队定律。

    上周的政策让我加深了对 [流动性] 这个词语的理解：能流通的才有流动性。中国人喜好储蓄，但储蓄就意味着货币流动性的缺失；宽松的货币政策与储蓄的理念相悖。换句话将就是：勤劳不会致富，储蓄使积攒的财富丢失。 穷人储蓄不多，更在意的是是否有工作，没工作就没饭吃；富人可以钱生钱，宽松的货币政策会使他们的钱更多。钱生钱，本身就需要一个符合经济运行的对这个世界的认知，与自身的诸如勇气之类的品质。或许一个合格的社会保障体制是能够令所有人享受宽松货币政策与经济发展带来的收益吧，而不是强调储蓄与保险。

    风险、流动性、收益是不可能满足的三角。在资金量多的情况下，与其 all-in 一个股票，不妨考虑下配置债券与期权，通过期权撬动更高的收益，通过债券对冲风险。另一方面，或许鸵鸟政策符合人性，同样是杠杆，面对房子多数人愿意去加——房价下跌不挂牌就不知道下跌，股票下跌是实实在在能感受得到的。
